Layer 1 — IPIN™

Every Government Contract.
Tracked & Structured.

Built in-house — not a data aggregator. A three-stage Python pipeline processes GeM contracts every day, enriched by a GPT-4 Agentic AI layer that classifies, maps, and scores each record.

Stage 1 — Discovery
GeM Contract Collector
The Pipeline Ops Agent automatically initiates a daily retrieval process that queries the GeM public contracts interface for contracts published during the previous day's date range. The process runs autonomously without manual intervention, dynamically determines the required date window, retrieves the available contract records, and passes the data to the downstream ingestion, validation, transformation, and storage pipeline. .
Daily · Automated
Stage 2 — Retrieval
PDF Download & Storage
Each contract PDF is downloaded and stored as an unaltered source-of-truth copy. Failed downloads are automatically retried; persistent failures escalate to the Data/Engineering team.
Python · Parallel Workers
Stage 3 — Enrichment
AI Parsing & Enrichment
Parallel workers extract structured fields from each PDF. A GPT-4 Enrichment Agent then maps product categories, resolves OEM associations, deduplicates buyer/seller entities, and assigns a confidence score — routing low-confidence records to manual review.

Powered by Agentic AI

Three AI Agents. Zero Manual Ops.

A three-tier GPT-4 Agentic AI layer sits across the pipeline, the data, and the user interface — replacing manual operations with autonomous, self-correcting intelligence.

Tier 1 — Pipeline Agent
Pipeline Operations Agent

Autonomously monitors and orchestrates the daily contract retrieval → document download → data extraction workflow. Manages scheduling, execution, health monitoring, retries, exception handling, and recovery, eliminating the need for manual pipeline operations..

Triggers daily scraper run without manual initiation
Detects & classifies failures; retries automatically
Delivers daily health summary to Engineering team
Escalates unresolved failures with structured report
Tier 2 — Enrichment Agent
Data Enrichment Agent

Reads every parsed contract record and enriches it with AI-generated category classifications, entity mappings, and confidence scores.

Maps contracts to standardised IPIN™ category taxonomy
Identifies and maps OEM/brand from product descriptions
Deduplicates buyer & seller entities across naming variants
Routes low-confidence records to manual review queue
Tier 3 — Conversational Interface
Conversational Intelligence

Internal teams ask procurement questions in plain English and get instant, cited answers from the IPIN™ database. No dashboards. No SQL.

Translates natural-language queries into database queries
Cites the exact contracts forming each answer
Handles ambiguous queries with a clarifying question
Strictly scoped to IPIN™ data — no hallucinated answers

What Makes This Defensible.

Each layer of the framework strengthens every other. The longer a client engages, the harder it becomes for a single-layer competitor to match what Indiquad delivers.

01
Proprietary Data Pipeline

02
Network Effects

More OEM clients → more GBO transactions → richer IPIN™ data → sharper IPAN™ targeting → more resellers onboarded → larger IPTS™ scoring pool → more NBFC subscribers. Every layer growing makes every other layer more valuable.

03
Three-Sided Ecosystem

Indiquad sits at the centre of OEMs, resellers, and lenders. Any single-layer competitor touches only one side. Indiquad touches all three simultaneously — making switching costly and cross-sell natural.

04
Compounding Intelligence

Unlike consultant knowledge that leaves when the engagement ends, IPIN™'s intelligence compounds in a database that gets sharper with every contract tracked, every engagement completed, and every reseller profiled.
